Daily Workflows

Follow these checklists every day to keep production, packing, and compliance running smoothly.

🌅 Morning — All Staff
  • Open Dashboard
  • Check Low Raw Materials alert
  • Check Open Orders count
  • Check Due in 48hrs alert
  • Download Production Schedule PDF
  • Start new production batches if needed
☀️ During the Day
  • Import Pastel PDF (Sales Orders)
  • Review incoming Supplier Receipts
  • Open Packing Queue — pack all Ready orders
  • Monitor Stock on Floor levels
  • Record any new production batches
  • Check Shop POS for walk-in sales
🌙 End of Day
  • Pack any remaining orders
  • Upload HACCP documents on Dashboard (one upload covers all today's orders)
  • Generate Final Packing Report
  • Check Delivery Schedule for tomorrow
  • Quick stock count if needed

Module 02

Production

Create new baking batches. Each batch uses raw materials from stock and adds finished biscuits to Stock on Floor.

📍 Left sidebar → Production — or click + New Production on the Dashboard
🆕 Starting a New Production Batch
1
Click "+ New Production" or go to Production in the sidebar
You can start from the Dashboard quick-action button (green) or via the left sidebar menu.
2
Select the Product (Flavour)
Choose from the dropdown: Choc Crunch, Coconut, Crunchie, Choc Chip, Ginger, Shortbread, etc. Do not create a separate batch for Assorted/Mix — the system handles flavour distribution automatically.
3
Enter Batches Produced
The number of individual batches baked (e.g. 5, 10, 20). Each batch is a fixed recipe quantity.
4
Set Production Date
Defaults to today. Change only if you're entering a batch that was baked on a previous day.
5
Check Expiry Date
The system auto-fills this based on product type. Verify it is correct before saving.
6
Select Shift
01 Day or 02 Night. Choose whichever shift baked this batch.
7
Select Production Supervisor
Choose the supervisor who oversaw this baking run. This is required for HACCP traceability.
8
Note the Baking Batch Number (auto-generated)
The system generates this automatically in the format ddmmyy-shift-sequence. You do not need to type it — but note it for reference.
9
Add QA Notes (optional)
Record any quality observations, deviations, or issues with this batch. These appear in the Final Packing Report.
10
Select Ingredients & Lots Used
The system pulls in the required ingredients for the selected product automatically. You must select the correct lot numbers for each ingredient — this is critical for traceability. Always use the oldest lot first (FIFO).
11
Click "Save Production Batch"
The system will ask "Are you sure?" — confirm to save. The finished product stock immediately updates on Stock on Floor.
📅 Generating a Production Schedule PDF

The Production Schedule is a professional colour-coded PDF printed and given to bakers each shift.

1
Go to Past Production or the Sales Orders screen
2
Click the green "Schedule PDF" button (top right)
3
Select which orders / products to include
4
Download and print the PDF for bakers
💡
Always select accurate lot numbers — if you choose the wrong lot, the traceability chain breaks. The system will show which lots are available for the selected ingredient.
💡
After saving, the finished stock will appear automatically in Stock on Floor. You don't need to enter it separately.

Module 05

Sales Orders

All customer orders live here. You can import from Pastel automatically, create phone orders manually, or enter new sales orders directly. Each order is either Standard or Custom — understanding this difference is essential.

📍 Left sidebar → Sales Orders
📌 Standard vs Custom Orders — What's the Difference?

Every order line — whether entered via Phone Order, New Sales Order, or Pastel import — is classified as either Standard or Custom. This controls how the product dropdowns work.

TypeWhen to UseHow It Works
StandardThe customer wants a pre-defined pack from the standard product range — e.g. a 2.9kg Family Box of Shortbread, or a 1.5kg Bucket of Choc Chip.Three cascading dropdowns appear: first select Flavour, then Category (Box / Packet / Bucket / Tube), then Pack Type (the specific size). The system filters each dropdown based on your previous selection so you can only choose valid combinations.
CustomThe customer wants something outside the standard range — a non-standard weight, a special assortment, or a bespoke pack configuration.Free-form entry — enter flavour, pack description, quantity and kg manually. No dropdown filtering applies.
💡
Always try Standard first. If the exact pack the customer wants appears in the Flavour → Category → Pack Type cascade, use it — this ensures the correct Pastel item code is linked and the packing schedule generates correctly. Only use Custom if the pack genuinely doesn't exist in the standard range.
⭐ Method 1: Import Pastel PDF (Recommended — Fastest)

The fastest and most automated method. Use every day when you receive the Pastel order export.

1
Click "↑ Import Pastel PDF" (top right button)
2
Select and upload the Pastel PDF file
The file must use a semicolon (;) as the delimiter. If the import fails, this is the most common cause — re-export from Pastel with the correct setting.
3
The AI automatically reads and processes the file
It creates Sales Orders, matches Pastel item codes to Standard pack types, calculates what stock is already available, plans any additional production runs needed, and suggests exact quantities to produce.
4
Review the results on screen
Check the proposed orders and production plan. Make edits if needed before confirming.
5
Confirm — everything is created automatically
New orders appear in Sales Orders. Orders with sufficient stock move to the Packing Queue. Required production appears in the Production module.
📞 Method 2: Phone Order

For orders received by phone or WhatsApp that aren't coming through Pastel.

1
Click "Phone Order" (top left button)
2
Search for the customer
Type the customer name, account number, or phone number. For Shop orders, the phone number is the key matching field — make sure it is entered correctly.
3
Enter Order Date and Delivery Date
4
Select Standard or Custom for each order line
For Standard: select Flavour → Category → Pack Type from the cascading dropdowns. For Custom: enter the details manually.
5
Enter quantity — kg calculates automatically
For Standard orders the total kg is calculated from the pack type weight. For Assorted/Mix packs the system auto-populates the five contributing flavours (Shortbread, Ginger, Choc Chip, Coconut, Crunchie) using FIFO stock.
6
Add Notes if needed
Delivery instructions, special requests, or WhatsApp message content.
7
Save the order

4

Module 06

Packing Queue

Every order that is ready to be physically packed appears here. This is where the packing team works from each shift. The queue shows real-time stock availability, pre-packed stock indicators, and partial packing status.

📍 Left sidebar → Packing Queue
📋 Understanding the Queue
Indicator / BadgeMeaning
Total in QueueTotal number of orders waiting to be packed, including those without enough stock yet
Ready to PackOrders where all required stock is currently available on the floor — always start with these
In QueueOrder is waiting — either stock not yet available, or packing hasn't started
In PackingPacking has started but is not yet complete — the order is locked to prevent it being double-scheduled on another packing run
Pre-packed indicatorA coloured visual indicator on the order card showing that some or all stock for this order has already been floor-packed and is ready to allocate. The system will use this FIFO stock first.
kg shown on cardTotal weight of the order — gives the packing team an idea of shift workload
📦 Downloading the Packing Schedule PDF
1
Click "↓ Packing Schedule" at the top of the Packing Queue screen
2
The PDF generates and downloads automatically
This is a two-page document. Page 1 is the shift summary. Page 2 is the detailed per-order breakdown showing exactly which production batch each line must come from. Print both pages and hand to the packing supervisor at the start of each shift.
✅ How to Pack an Order (Standard Flow)
1
Click "Pack This Order" on the order card
The order status immediately changes to "In Packing" — this prevents anyone else from scheduling the same order simultaneously.
2
Review the packing details screen
The system shows the auto-generated Packing Batch Number (format: PACK[ddmmyy]-01), packing date, shift, and supervisor/packer fields.
3
Confirm batch allocation
The system auto-selects production batches using FIFO — oldest stock first. If pre-packed floor stock exists for this order, it is allocated first before drawing from bulk production batches. For mixed/assorted packs (buckets), the system distributes across the five flavours automatically.
4
Complete QA checks
Tick all six quality check items: Colour Good, Taste Good, Label Correct, Texture Good, Weight Correct, Seal Integrity Correct.
5
Save / Complete
Stock on Floor is reduced immediately. The order moves to the Delivery Schedule (Retail) or shows "Ready for Collection" (Shop).
🔄 Partial and Split Packing

If an order is too large to complete in one shift, or if stock runs short mid-pack, you can pack it in stages across multiple shifts or days.

ConceptHow It Works
Partial quantity selectionOn the packing screen, you can select fewer units than the full order quantity for each line — e.g. pack 30 of the 50 required boxes now.
In Packing statusOnce any packing starts on an order, it shows "In Packing" — it will not appear as a fresh order on the next packing schedule. This prevents double-counting.
Packed / Remaining balanceThe order card updates to show cumulative totals: how many units have been packed across all shifts so far, and how many remain outstanding.
Carry-over to next shiftThe outstanding quantity automatically carries over. When enough stock is available for the remaining units, the order becomes ready to pack again.
Assorted/Mix linesAssorted lines are always treated as a single atomic unit — you cannot partially pack just one flavour within an Assorted line. The entire assorted allocation for that partial quantity is packed together.
💡
Always complete the physical packing before saving in the system. The system records what you enter as packed — if you save a quantity that wasn't actually packed, the stock figures will be wrong.
📦 Pre-Packed Floor Stock

Pre-packed stock is biscuits that the packing team has already packed into boxes, buckets, or packets before a specific customer order was assigned — for example, during a quiet period to get ahead of demand.

AspectDetail
Where it's recordedPre-packed stock is recorded in the Pre-Packed Stock module (accessible from the Packing Queue). Staff enter the flavour, pack type, quantity packed, and which production batch was used.
How it appears on order cardsIf pre-packed stock is available for a flavour/pack type an order needs, the order card shows a visual indicator. This tells the packing team that ready stock exists and doesn't need to be packed from scratch.
FIFO allocationWhen the packing schedule is created, pre-packed stock is allocated to orders automatically in FIFO order — oldest pre-packed stock is used first.
Packing schedule notationOn the Packing Schedule PDF, a line that uses pre-packed stock shows: "PRE-PACKED [X units] from Batch PACK… — PACK [Y units] still required". Only pack the remaining Y units physically — the X units are already done.
TraceabilityPre-packed stock retains full traceability to the original production batch and raw ingredients, exactly as if it had been packed directly against the order.
⚠️
HACCP documents are not uploaded here. Upload them once at end of day via the Dashboard. One upload links to all orders packed on that day.
Module 07

Delivery Schedule

Once orders are packed, they appear here ready for dispatch. Retail orders go to delivery; Shop orders go to collection. This is the driver's daily work list.

📍 Left sidebar → Delivery Schedule
🚛 How Delivery Works
1
All packed Retail orders appear here automatically
Shop (collection) orders do not appear in the Delivery Schedule — they show "Ready for Collection" status on the Sales Orders screen instead.
2
Orders are sorted oldest-first by required delivery date
Orders overdue or due today appear at the top. Always dispatch the oldest-dated orders first.
3
Driver loads the vehicle and departs
The driver works through the delivery list in order. Each stop's customer name, address, and order details are shown.
4
Mark each order as Delivered once dropped off
As orders are marked Delivered they move to Past Orders and are removed from the active Delivery Schedule.
5
"All Delivered!" confirms the queue is clear
When all deliveries are complete the screen shows the "All Delivered!" message. The day's delivery run is done.
🏪 Shop Orders — Collection Flow
StepWhat Happens
Order placedShop order created in Sales Orders — always linked to the customer's phone number
Packing completeWhen the packing upload is scanned in, the system matches the order via phone number and automatically sets status to "Ready for Collection"
Customer arrivesStaff check Sales Orders — find the order showing "Ready for Collection" — hand over the goods
Mark collectedUpdate the order status to Collected/Delivered to close it out and move it to Past Orders
⚠️
Never add a Shop order to a delivery route. Shop orders are always collection-only. If a Shop order appears on the Delivery Schedule in error, check the order type — it may have been entered as Retail by mistake. Contact Admin to correct it.
💡
If the Delivery Schedule seems to be missing an order that should be there, check the Packing Queue first — the order may still be "In Packing" status, which means packing hasn't been fully confirmed yet.

📋 Fields for Each Ingredient Line

  • Cost per KG REQUIREDUnit cost — used for stock valuation and profitability reports

🕐 Viewing Past Receipts

Reports & Analytics

Support

Troubleshooting

Common issues and how to fix them. Try these before contacting your system administrator.

Pastel PDF import fails — what's wrong?
The most common cause is the file delimiter. Pastel exports must use a semicolon (;) as the delimiter, not a comma. Open the file in Notepad to check — if values are separated by commas, re-export from Pastel with the correct setting. Also ensure the file is a genuine Pastel export, not a manually created CSV.
Stock levels aren't updating after an upload
After uploading a stocktake, you must click "Confirm & Import" on the results screen. If you closed the window before confirming, the upload did not complete. Repeat the upload and confirm. Then refresh the Stock on Floor or Raw Materials screen.
An ingredient isn't showing in the supplier receipt dropdown
Ingredients are filtered by the selected supplier. First check that you have selected the correct supplier for this receipt. If the ingredient genuinely belongs to that supplier but still doesn't appear, the supplier link on the ingredient record may be broken — contact Admin. The fix requires linking by supplier ID, not by typing the supplier name as text.
Pack types or products are missing from dropdowns
Contact Admin to run the pack type fix functions. Missing pack types usually appear after a data migration or if a new product was added to the system without the correct configuration.
Costs look wrong in reports
Cost data comes from two sources: the Ingredients table (raw material cost per kg) and the ProductCodeMappings table (product pricing). Ask Admin to update the relevant entry. Do not adjust costs directly in a supplier receipt unless the unit price was genuinely different for that delivery.
The HACCP document isn't showing on an order's Final Packing Report
The HACCP document links to the date it was uploaded on the Dashboard. If the packing date and the upload date don't match, the link won't form. Go to the Dashboard, select the correct historical date, and upload the document for that date.
Stock on Floor shows OUT but we have stock in the warehouse
The system only tracks stock that has been formally recorded — either via a production batch save or a stocktake upload. Manually counted stock that wasn't entered won't appear. Upload a stocktake via Data Imports to bring the figures in line.
I can't find a menu item I should have access to
Your account role determines which menu items are visible. If you need access to a module your role doesn't include, ask Admin to update your role in User Management. Some items (e.g. User Management itself) are Admin-only by design.
The system is running slowly or looks outdated
Press F5 (Windows/Linux) or Cmd+R (Mac) to force a browser refresh. The Dashboard refreshes automatically but other screens may need a manual refresh. If the issue persists, clear your browser cache (Ctrl+Shift+Delete) and try again.
Two users edited the same supplier receipt at the same time and the quantities are wrong
Concurrent edits by two users simultaneously can cause conflicts. One user's changes may overwrite the other. Always check the most recent supplier receipt after any edits if two people have been in the system at the same time. Contact Admin if the quantities need correcting via a backend reconciliation.

Important Business Rules

These rules are built into the system and reflect how Crunch 'n Munch operates. Understanding them prevents errors.

RuleDetail
Assorted / Mix — never batch separatelyThe system distributes Assorted KG across 5 flavours automatically: Shortbread, Ginger, Choc Chip, Coconut, Crunchie. Never create a separate "Assorted" production batch.
FIFO is mandatoryStock is always packed oldest-batch-first. The system enforces this in the Packing Queue. Never override this order without management approval.
Batch number formatAuto-generated as ddmmyy-shift (e.g. 220626-01 = 22 June 2026, Day shift). Never manually create or alter batch numbers.
HACCP upload — once per dayOne upload at day-end covers all packed orders for that day. Uploading more than once for the same date overwrites the previous upload.
Shop orders — always collectionShop-type orders are never delivered. They must never appear on a delivery route. Status is "Ready for Collection."
White Bread Flour — zero-ratedNo VAT applies to White Bread Flour purchases. The VAT line will show R 0.00 — this is correct.
Lot numbers — always captureEvery supplier ingredient must have a lot number for traceability compliance. "Fill in later" is a temporary measure only — complete it the same day.
Save confirmation always appearsThe system asks "Are you sure?" before every save. This is intentional — always review the screen before confirming.
Supplier links — by ID not textIngredient-to-supplier links in the system use database IDs. Typing a supplier name in a text field does NOT create a link. Use the dropdown selector.
Concurrent edits — be carefulIf two users edit the same record simultaneously, the last save wins. Coordinate with colleagues before editing shared records.

📌 Key Things to Know About the Production Slip
💡
Assorted lines: When you see "Assorted" as a flavour on a Production Slip, this means the customer wants a mix of flavours in that pack. The system automatically distributes the kg across Shortbread, Ginger, Choc Chip, Coconut, and Crunchie. Never bake a separate "Assorted" batch — just ensure all five contributing flavours have enough stock.
💡
Multiple lines of the same pack type: A customer may order the same pack size in multiple flavours — e.g. six separate lines of 250g Box (one per flavour). Each line is packed and tracked separately even though the pack type is the same.
💡
Choc Crunch appears separately: Choc Crunch is always listed as its own line and never forms part of an Assorted mix. It uses a different box type (400g Box) and a unique product code.
🔄 How the Production Slip Relates to the Packing Schedule

The Production Slip is a quick customer-level summary. The Packing Schedule (Document A) is the shift-level document that tells the packing team exactly which production batches to use and requires sign-offs. Both documents come from the same underlying sales order — the Production Slip is for reference, the Packing Schedule is for action.
